Какую библиотеку используем для свечного графика?

Каждая из трёх даёт разный баланс между качеством рендера, весом и усилиями на интеграцию

A — lightweight-charts (TradingView)

Канвас-рендер, 45KB gzip. Профессиональный вид «из коробки» — кросс-хэр, скролл, зум, OHLCV-тултип. Используется на TradingView.com.

Плюсы

  • Лучший UX среди трёх
  • Встроенный зум/скролл
  • Отрисовка 10K свечей без лагов
  • Автоматический crosshair

Минусы

  • Новая зависимость (~45KB)
  • Не React-native (imperative API)
  • Придётся обернуть в useEffect

B — react-financial-charts

SVG-рендер на базе d3. React-компонент с деклараттивным API. Активно поддерживается, есть примеры с RSI/Bollinger.

Плюсы

  • Декларативный React-API
  • Встроенные индикаторы (RSI, Bollinger, MACD)
  • Отдельные панели из коробки

Минусы

  • Тяжёлее: ~200KB (d3 + lib)
  • Устаревший стиль SVG (медленнее на больших данных)
  • Слабая TypeScript-поддержка

C — Recharts (уже есть) + кастомная форма

Рисуем свечи через customShape в <Bar>. Нет новых зависимостей — но придётся вручную делать зум, кросс-хэр, RSI-панель.

Плюсы

  • Нулевые новые зависимости
  • Стиль совпадает с остальными графиками

Минусы

  • Большой объём ручной работы
  • Нет нативного зума/скролла
  • RSI-панель — отдельный компонент, синхронизация вручную
  • Производительность хуже при >500 свечах

💡 Рекомендация: A — lightweight-charts. Добавляет ~45KB, но даёт TradingView-уровень UX без тонн ручного кода.
